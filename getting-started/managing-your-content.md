# Managing Your Content

Content in AIBC moves through a lifecycle from creation to publication. Understanding this workflow helps you manage content effectively and maintain quality control.

## Content Lifecycle

Content typically moves through these statuses:

1. **Draft** → Initial creation, not yet ready
2. **Needs Review** → Waiting for approval
3. **Approved** → Ready to publish
4. **Scheduled** → Set to publish at specific time
5. **Published** → Live on platform(s)

## Content Statuses Explained

### Draft

**What it means:**
- Content has been created but isn't finalized
- Not scheduled for publishing
- Can be edited, regenerated, or deleted
- Doesn't count toward publishing limits

**When content is in draft:**
- Just created manually
- Generated but not yet reviewed
- Saved for later editing
- Waiting for additional information

**What you can do:**
- Edit content
- Regenerate with new brief
- Add media or images
- Schedule for publishing
- Delete if not needed

### Needs Review

**What it means:**
- Content is ready for review
- Waiting for approval before publishing
- Appears in Inbox for review
- Requires action to proceed

**When content needs review:**
- Auto-generated content (if approvals enabled)
- Manually created content
- Regenerated content
- Content requiring approval based on settings

**What you can do:**
- **Approve**: Move to scheduled/published
- **Request changes**: Send back for editing
- **Reject**: Remove from queue
- **Edit**: Make changes before approving
- **Comment**: Add feedback for team

### Approved

**What it means:**
- Content has been approved
- Ready to publish
- Can be scheduled or published immediately
- Meets quality standards

**What you can do:**
- **Schedule**: Set specific publish date/time
- **Publish now**: Publish immediately
- **Edit**: Make final adjustments
- **Reschedule**: Change publish time

### Scheduled

**What it means:**
- Content is set to publish at specific time
- Appears in Calendar view
- Will publish automatically (if auto-publish enabled)
- Can be edited or rescheduled before publishing

**What you can do:**
- **View in Calendar**: See scheduled date/time
- **Reschedule**: Change publish time
- **Edit**: Make changes before publishing
- **Unschedule**: Remove from schedule
- **Publish now**: Publish immediately instead

### Published

**What it means:**
- Content is live on platform(s)
- Appears in published content library
- Can track performance in Analytics
- Can be edited or deleted (platform-dependent)

**What you can do:**
- **View on platform**: See live content
- **Track performance**: Check analytics
- **Edit** (if platform allows): Make changes
- **Delete**: Remove from platform
- **Repost**: Share again (if appropriate)

## Managing Content in Content Hub

### Viewing Content

**Filters:**
- **Status**: Draft, Needs Review, Approved, Scheduled, Published
- **Platform**: Twitter, LinkedIn, Instagram, etc.
- **Type**: Post, Thread, Reel, Blog, etc.
- **Date**: Created date, scheduled date, published date
- **Tags**: Custom tags you've added

**Views:**
- **List view**: Detailed list with all information
- **Grid view**: Visual cards (coming soon)
- **Calendar view**: See in Calendar page

### Bulk Actions

**Select multiple content items:**
1. Check boxes next to content
2. Click **"Bulk Actions"**
3. **Available actions:**
   - **Approve**: Approve selected items
   - **Schedule**: Schedule all for same time
   - **Delete**: Remove selected
   - **Add tags**: Tag multiple items
   - **Move to campaign**: Assign to campaign
   - **Export**: Download selected

### Searching Content

**Search functionality:**
- **Text search**: Search content text
- **Filter search**: Use filters to narrow down
- **Tag search**: Find by tags
- **Date range**: Filter by date

## Content Workflows

### Automatic Publishing Workflow

**If auto-publish is enabled:**

1. Content is generated → **Draft**
2. Content is reviewed (optional) → **Needs Review** (if enabled)
3. Content is approved → **Approved**
4. Content is scheduled → **Scheduled**
5. Content publishes automatically → **Published**

### Manual Review Workflow

**If approvals are required:**

1. Content is generated → **Needs Review**
2. Review in **Inbox**
3. **Approve** → Moves to **Approved**
4. **Schedule** → Moves to **Scheduled**
5. **Publish** → Moves to **Published**

### Editing Workflow

**To edit content:**

1. Find content in **Content Hub**
2. Click to open content details
3. Click **"Edit"**
4. Make changes:
   - Edit text
   - Replace images
   - Change platform
   - Adjust schedule
5. **Save** changes
6. Status may change based on settings

## Organizing Content

### Using Tags

**Add tags to content:**
1. Open content details
2. Find **"Tags"** section
3. Add existing tags or create new
4. Tags help organize and filter

**Tag best practices:**
- Use consistent naming
- Create tag hierarchy (e.g., "Campaign: Q1 Launch")
- Use descriptive tags
- Don't over-tag

### Using Campaigns

**Assign to campaign:**
1. Select content
2. Click **"Assign to Campaign"**
3. Choose existing or create new
4. Content is grouped by campaign

**Campaign benefits:**
- Group related content
- Track campaign performance
- Organize by project
- Filter by campaign

### Using Favorites

**Mark as favorite:**
1. Click star icon on content
2. Content is saved to favorites
3. Filter by "Favorites" to see all

**Use cases:**
- Save best-performing content
- Keep examples for reference
- Mark content to reuse
- Create inspiration library

## Content Quality Control

### Review Checklist

**Before approving content:**
- ✅ Matches brand voice
- ✅ On-topic and relevant
- ✅ No errors or typos
- ✅ Appropriate for platform
- ✅ Images/media are correct
- ✅ CTA is clear
- ✅ Links work (if applicable)

### Quality Standards

**Set quality standards:**
- Define what "good" content means
- Create review criteria
- Train team on standards
- Use Improve Feature for guidance

### Continuous Improvement

**Learn from content:**
- Track what performs well
- Identify patterns in winners
- Update Brand Kit based on learnings
- Refine voice and preferences

## Content Maintenance

### Regular Review

**Weekly tasks:**
- Review content in queue
- Approve or request changes
- Schedule upcoming content
- Check for duplicates

**Monthly tasks:**
- Review published content performance
- Archive old content
- Update content strategy
- Clean up drafts

### Content Cleanup

**Remove unnecessary content:**
- Delete old drafts
- Remove rejected content
- Archive published content
- Clean up test content

## Common Issues

### Content Stuck in Review

**Solution:**
- Check Inbox for pending reviews
- Approve or request changes
- Check if approvals are required
- Review approval settings

### Content Not Publishing

**Solution:**
- Verify content is scheduled
- Check platform connection
- Ensure auto-publish is enabled (if using)
- Check for errors in Calendar
- Verify platform permissions

### Too Much Content in Queue

**Solution:**
- Review and approve faster
- Enable auto-publish (if appropriate)
- Reduce content generation volume
- Use bulk actions for efficiency
- Set up approval workflows

### Content Quality Issues

**Solution:**
- Update Brand Kit
- Add better source materials
- Refine voice settings
- Use Improve Feature
- Provide better briefs when regenerating

## Best Practices
